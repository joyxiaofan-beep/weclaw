# 🖥️ 路径 A：终端模式（3 分钟上手）

> 零配置，纯命令行交互。
> 龙虾的 AI 大脑、人脉记忆、上下文理解一个不少。

## A-1：安装（1 分钟）

```bash
# 克隆项目
git clone <你的仓库地址> && cd weclaw

# 创建虚拟环境（推荐）
python3 -m venv .venv && source .venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

## A-2：配置 AI（1 分钟）

**方式一**：环境变量（最快）

```bash
export OPENAI_API_KEY=sk-xxxxxxxx
# 如果用 DeepSeek/Moonshot 等国产模型：
# export OPENAI_BASE_URL=https://api.deepseek.com/v1
# export OPENAI_MODEL=deepseek-chat
```

**方式二**：配置文件

```bash
cp config/config.terminal.yaml config/config.yaml
# 编辑 config/config.yaml，填入你的 api_key
```

## A-3：启动龙虾 🚀

```bash
python -m weclaw
```

你会看到：

```
    🦞 WeClaw — 终端模式

  龙虾已上线！在终端直接跟我对话。
  消息不会真正发出，你可以放心体验所有功能。

  💡 快速上手：
  • 直接打字 → 给龙虾下指令
  • @小王 数据好了 → 模拟小王回复了消息
  • quit → 退出
──────────────────────────────────────────────────
```

| 你输入 | 效果 |

# 🦞 路径 C：龙虾互联（5 分钟，零配置）

> 两只龙虾聊天！不需要公网 IP、不需要 ngrok、不需要改配置。
> 像微信加好友一样简单——每只龙虾有自己的「龙虾号」，加一次好友，永久在线。

## C-1：启动 Relay 中继服务器

```bash
# 在一个终端里启动 Relay（两只龙虾都连到这里）
python relay_server/server.py
```

你会看到：

```
🦞🌐 WeClaw Relay Server v2 启动中...
   地址: ws://0.0.0.0:8900
   最大在线: 2000
✅ Relay Server 已就绪，等待龙虾连接...
```

> 💡 如果你和朋友不在同一个局域网，需要把 Relay 部署到有公网 IP 的服务器上。
> 支持 Docker 部署：`cd relay_server && docker build -t weclaw-relay . && docker run -p 8900:8900 weclaw-relay`

## C-2：启动你的龙虾

```bash
# 另开一个终端
python -m weclaw
```

首次启动会自动生成你的**龙虾号**（类似微信号，重启不变）：

```
🆕 首次启动，已生成龙虾号: lobster_a3f7b2c1

✅ Relay 已连接！

   🦞 龙虾号: lobster_a3f7b2c1
   🎫 加好友码: #3847（一次性，10分钟有效）

   把加好友码发给朋友，对方输入:
   龙虾加好友 #3847
   加为好友后，以后每次上线自动连接！
```

**把加好友码 `#3847` 发给你的朋友。**

> 💡 龙虾号是永久身份，加一次好友就行。加好友码是一次性的（用完即毁，像面对面加微信）。

## C-3：朋友启动龙虾 & 加好友

朋友在自己电脑上：

```bash
# 如果 Relay 在你的电脑上，朋友需要指定 Relay 地址
export RELAY_URL=ws://你的IP:8900
python -m weclaw
```

朋友输入：

```
龙虾加好友 #3847
```

双方都会看到好友添加成功的消息：

```
🦞🤝✅ 好友添加成功！

🦞 🦞 朋友的龙虾
👤 主人: 朋友
🆔 龙虾号: lobster_e5d9f1a0
🎯 能力: 代主人传话, 查看主人是否方便

现在你可以用 "龙虾传话 朋友 <消息>" 来传话了！
以后重启自动连接，不需要再加好友。
```

```
你 ▸ 龙虾传话 朋友 明天下午有空吗

🦞→🦞 传话成功!
📤 发给 朋友 的龙虾: 「明天下午有空吗」
📥 对方龙虾回复: 「已收到，会转告朋友」
```

**🎉 恭喜！两只龙虾已经在聊天了！**

### 龙虾互联可以做什么

| 你输入 | 效果 |
|--------|------|
| `龙虾加好友 #1234` | 通过加好友码添加另一只龙虾（一次性） |
| `龙虾号` / `我的龙虾号` | 查看你的龙虾号 + 加好友码 |
| `龙虾传话 老王 内容` | 给老王的龙虾传话 |
| `龙虾回 老王 内容` | 回复老王龙虾之前的消息 |
| `龙虾通讯录` | 查看好友列表 |
| `龙虾信任 老王` | 信任某只龙虾（允许传话） |
| `龙虾删除 老王` | 从好友列表移除 |

> 💡 **龙虾号是永久的**——存在本地 SQLite，重启不变。加一次好友，以后每次上线自动互认。
> 💡 加好友码是**一次性**的——加好友成功后自动失效，10 分钟未使用也会过期。
> 💡 好友列表持久化到 SQLite，重启不会丢失。

## ⚠️ Day 1 常见问题

### Q: "龙虾加好友"失败，提示加好友码无效？
> - 加好友码是**一次性**的，用过就失效。让对方重新查看加好友码（输入 `龙虾号`）
> - 加好友码有 10 分钟有效期，过期需要对方重启获取新码
> - 确认 Relay Server 正在运行：`python relay_server/server.py`
> - 确认你的龙虾已连接 Relay（启动时会显示龙虾号和加好友码）
> - 如果朋友在其他网络，确认 `RELAY_URL` 指向了正确的 Relay 地址
> - 注意格式：`龙虾加好友 #1234`（带 # 号）

### Q: Relay 连接失败？
> - 确认 Relay Server 已启动：终端里运行 `python relay_server/server.py`
> - 如果 Relay 在远程服务器，检查防火墙是否放行了 8900 端口
> - 用 `--no-relay` 启动可以跳过 Relay，使用 HTTP 直连模式

### Q: AI 回复很慢或报错？
> - 检查 `ai.api_key` 是否正确
> - 如果用第三方转发，检查 `ai.base_url`
> - 龙虾会自动重试 3 次，如果还失败会通知你"AI 暂时不可用"

### Q: 能用国产模型吗？
> 能！只要是 OpenAI 兼容格式的 API 都行。配置 `ai.base_url` 指向你的 API 即可，比如：
> ```yaml
> ai:
>   base_url: "https://api.deepseek.com/v1"
>   api_key: "你的 DeepSeek key"
>   model: "deepseek-chat"
> ```
