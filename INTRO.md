# WeClaw：一只跑在终端里的社交代理龙虾

> 开源 MIT | Python 3.10+ | 3 分钟可上手

---

## 一、Why — 为什么会做这样一个东西

### 1.1 起点

日常工作中协调沟通的频率很高——跟不同的人对齐进度、催数据、约时间、同步信息。这些事情并不复杂，但反复做很耗精力。每次都要想措辞、翻聊天记录找上下文、记住谁擅长什么，这些零散的认知负担加起来并不小。

去年在接触各类 AI 工具后，我开始想：能不能把这些重复性的沟通动作交给一个 Agent，让它帮我起草消息、记住联系人的特点、追踪待回复的事项？不是替代社交本身，而是替我处理其中机械的部分。

### 1.2 已有工具的不足

试了一些方案后发现几个问题：

**ChatGPT / 通用 AI 助手**——每次对话都得重新交代背景。上一轮刚聊完的人和事，切个窗口就忘了。它不认识你的同事，不知道你上周跟谁约了什么，也不会主动帮你追踪回复。

**各类聊天机器人**——大多还是"你问它答"的交互模式。能帮你写一段话，但不能帮你"办事"——不能代你跟踪一件事情的进展，不能在你忘了的时候提醒你，也不能跟别人的 Agent 直接通信。

**数据隐私**——联系人信息和沟通内容是比较敏感的数据，放到云端总归不太放心。

### 1.3 所以做了 WeClaw

WeClaw 是一只跑在终端里的龙虾。龙虾有两只钳子——一只收、一只发——天生就是通信员的料。

它目前能做的事情：

- **代你起草消息**——你说意图，它组织措辞，确认后才发送
- **接收消息并提炼摘要**——帮你过滤噪音、抓重点、建议下一步
- **记住联系人的特点**——每次交互自动学习，越用越懂你的人脉
- **追踪待回复事项**——超时自动提醒，不会漏掉重要的事
- **上下文记忆**——知道"他"是谁、"那件事"是什么
- **龙虾互联（Claw-to-Claw）**——你的龙虾可以跟朋友的龙虾直接对话

坦率说，目前还是一个比较早期的项目。核心能力跑通了，但离"好用"还有距离。先把它开源出来，也是希望看看大家的想法。

## 二、What — WeClaw 做了什么

### 2.1 核心能力

WeClaw 的定位不是聊天机器人，而是一个**帮你办事的社交代理**。区别在于：

| | 通用 AI 助手 | WeClaw |
|---|---------|--------|
| **交互** | 你问它答 | 你说意图，它替你执行 |
| **记忆** | 单次会话 | 持久化联系人画像 + 对话上下文 |
| **通信** | 只有人→AI | 支持 Agent→Agent（龙虾互联） |
| **数据** | 存云端 | 存本地（YAML + SQLite），完全可控 |
| **追踪** | 无 | 自动追踪待回复消息，超时提醒 |
| **信任** | 无 | 渐进式信任体系（0→100 分） |

### 2.2 架构概览

```
你 (终端)
  │
  ▼
┌─────────────────────────────────────────────┐
│           WeClaw Core Brain                 │
│                                             │
│  ┌──────────┐ ┌──────────┐ ┌─────────────┐ │
│  │ 联系人    │ │ AI 对话  │ │ 龙虾互联    │ │
│  │ 记忆     │ │   层     │ │ (C2C)      │ │
│  │ (YAML)   │ │ (OpenAI) │ │ WebSocket  │ │
│  └──────────┘ └──────────┘ └─────────────┘ │
│  ┌──────────┐ ┌──────────────────┐          │
│  │ 超时追踪 │ │ 对话上下文       │          │
│  │ (SQLite) │ │ (最近 N 轮→AI)  │          │
│  └──────────┘ └──────────────────┘          │
│  ┌──────────────────────────────────┐       │
│  │  意图解析 + 快速路由              │       │
│  └──────────────────────────────────┘       │
└─────────┬────────────────────┬──────────────┘
          │                    │
     终端交互             龙虾互联
      (CLI)          (Relay / HTTP)
                          │
                   🦞 A ↔ 🦞 B
```

核心模块拆开看：

- **联系人记忆**——用 YAML 存储每个联系人的画像，AI 自动学习并更新特征标签
- **对话上下文**——用 SQLite 保留最近的对话轮次，注入 AI 时带上历史，支持代词消解
- **超时追踪**——发出消息后自动记录，超过设定时间未回复就提醒你
- **龙虾互联**——通过 Relay 中继服务器实现龙虾间 WebSocket 通信，不需要公网 IP

### 2.3 龙虾互联是怎么回事

这是 WeClaw 比较有意思的一个设计——两只龙虾可以直接对话。

每只龙虾有一个**龙虾号**（永久身份，像微信号）和一个**加好友码**（一次性，像面对面扫码）。加一次好友后永久互认，重启不丢。

通信走 Relay 中继服务器——Relay 只做路由转发，不存储任何消息内容。两只龙虾之间用 HMAC-SHA256 签名验证身份，防止消息伪造。

龙虾互联还有一个**渐进式信任体系**：

| 分数 | 等级 | 含义 |
|------|------|------|
| 0 | 陌生人 | 不理你 |
| 5 | 被引荐 | 朋友的朋友介绍的 |
| 10 | 已握手 | 完成了握手协议 |
| 50 | 好友 | 加过好友码 |
| 70+ | 已信任 | 可以传话、转发消息 |
| 100 | 完全信任 | 最高权限 |

信任是逐步建立的，不是加了好友就能做所有事。这个设计参考了现实世界的社交逻辑——信任需要时间和背书。

## 三、How — 具体怎么用

### 3.1 安装（1 分钟）

**方式一：pip 安装**

```bash
pip install weclaw
```

**方式二：从源码运行**

```bash
git clone https://github.com/joyxiaofan-beep/weclaw.git
cd weclaw
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```

### 3.2 配置 AI 模型

WeClaw 支持所有 OpenAI 兼容格式的 API。你只需要有一个 API Key。

**最快方式：环境变量**

```bash
# OpenAI
export OPENAI_API_KEY=sk-xxxxxxxx

# 或者用国产模型（DeepSeek、Moonshot、通义等）
export OPENAI_API_KEY=你的key
export OPENAI_BASE_URL=https://api.deepseek.com/v1
export OPENAI_MODEL=deepseek-chat
```

**或者：配置文件**

```bash
cp config/config.terminal.yaml config/config.yaml
```

编辑 `config/config.yaml`：

```yaml
ai:
  api_key: "你的API Key"
  model: "gpt-4o"                # 或 deepseek-chat / moonshot-v1-8k
  base_url: null                 # 国产模型需要填对应的 API 地址

behavior:
  confirm_before_send: true      # 发消息前需要你确认
  tone: "auto"                   # 语气风格：formal / casual / auto
```

### 3.3 启动

```bash
# pip 安装方式
weclaw

# 源码方式
python -m weclaw
```

启动后你会看到：

```
    🦞 WeClaw — 终端模式

  龙虾已上线！在终端直接跟我对话。
  消息不会真正发出，你可以放心体验所有功能。

  💡 快速上手：
  • 直接打字 → 给龙虾下指令
  • @张三 内容 → 模拟张三回复了消息
  • quit → 退出
──────────────────────────────────────────────────
```

> 终端模式下消息不会真正发出，联系人数据和对话记录会持久化到 `data/` 目录。

### 3.4 基础操作：消息起草与发送

```
你 ▸ 帮我问一下数据报告好了没

🦞 龙虾
📝 消息草稿（给 数据组同事）：

「Hi～上周提到的数据报告进展怎么样了？方便的话今天发我看一下~」

💡 根据你的意图生成，语气自然友好
回复 "发送1" 确认 | "取消1" 取消 | "改1 新内容" 修改
```

你确认后消息才会发送。如果草稿不满意，可以用 `改1 换个正式的语气` 让龙虾重写。

### 3.5 模拟收到消息

用 `@` 前缀模拟对方回复：

```
你 ▸ @数据组同事 报告已经发到你邮箱了，有两个指标标了备注，你看看

🦞 龙虾
📨 收到消息：
  💬 "报告已经发到你邮箱了，有两个指标标了备注，你看看"

  📋 报告已发送至邮箱，两个指标有备注待确认
  ⚡ 建议查看邮箱后回复确认
  🧠 学到了：做事靠谱，会主动标注问题
```

龙虾会自动提炼摘要、给出建议，同时把对方的行为特征记到联系人画像里。

### 3.6 上下文记忆

龙虾会记住最近的对话上下文，支持代词：

```
你 ▸ 帮我跟他确认一下那件事

🦞 龙虾
📝 消息草稿（给 数据组同事）：

「报告收到了，两个标备注的指标我看一下，有问题再找你~」
```

龙虾知道"他"是刚才聊到的数据组同事，"那件事"是数据报告。

### 3.7 联系人搜索

```
你 ▸ 谁懂数据分析？

🦞 龙虾
🔍 在你的人脉中找到了：
  1. 数据组同事 — 擅长 Python/SQL，响应快
  2. BI 团队成员 — 做过多个数据看板，分析经验丰富
```

这些搜索结果来自龙虾在历次交互中积累的联系人画像。用得越多，画像越准。

### 3.8 待办追踪

```
你 ▸ 待办

🦞 龙虾
📋 你有 2 条待回复：
  1. 产品组 — 关于需求排期（已等 3 小时）
  2. 设计师 — 关于 UI 稿确认（已等 1 天）⚠️ 超时
```

### 3.9 全部指令速查

| 你输入 | 效果 |
|--------|------|
| `帮我问XX 内容` | 起草消息给 XX |
| `发送1` | 确认发送第 1 条草稿 |
| `取消1` | 取消第 1 条草稿 |
| `改1 新内容` | 修改第 1 条草稿 |
| `@XX 消息内容` | 模拟 XX 发来了消息 |
| `谁懂XXX` | 从人脉中搜索 |
| `待办` | 查看等待回复的消息 |
| `帮我跟他确认一下` | 上下文推断对象和事项 |
| `XX是数据组的，擅长Python` | 教龙虾记住联系人信息 |
| `quit` | 退出 |

## 四、龙虾互联教程（两只龙虾对话，5 分钟）

如果你身边有朋友也部署了 WeClaw，可以让两只龙虾直接对话。不需要公网 IP，不需要 ngrok。

### 4.1 启动 Relay 中继服务器

```bash
# 在一台机器上启动 Relay
python relay_server/server.py
```

```
🦞🌐 WeClaw Relay Server v2 启动中...
   地址: ws://0.0.0.0:8900
✅ Relay Server 已就绪，等待龙虾连接...
```

> 如果你和朋友不在同一局域网，需要把 Relay 部署到有公网 IP 的服务器上。
> Docker 一行搞定：`docker run -p 8900:8900 $(docker build -q relay_server/)`

### 4.2 启动龙虾

```bash
python -m weclaw
```

首次启动会自动生成龙虾号和加好友码：

```
🆕 首次启动，已生成龙虾号: lobster_a3f7b2c1

✅ Relay 已连接！

   🦞 龙虾号: lobster_a3f7b2c1
   🎫 加好友码: #3847（一次性，10分钟有效）

   把加好友码发给朋友，对方输入:
   龙虾加好友 #3847
```

把加好友码 `#3847` 发给朋友。

### 4.3 朋友加好友

朋友启动自己的龙虾后输入：

```bash
# 如果 Relay 在你的机器上
export RELAY_URL=ws://你的IP:8900
python -m weclaw
```

```
龙虾加好友 #3847
```

双方都会看到：

```
🦞🤝✅ 好友添加成功！

👤 主人: 朋友
🆔 龙虾号: lobster_e5d9f1a0

现在你可以用 "龙虾传话 朋友 <消息>" 来传话了！
以后重启自动连接，不需要再加好友。
```

### 4.4 龙虾传话

```
你 ▸ 龙虾传话 朋友 明天下午有空吗

🦞→🦞 传话成功！
📤 发给朋友的龙虾: 「明天下午有空吗」
📥 对方龙虾回复: 「已收到，会转告」
```

### 4.5 龙虾互联指令

| 你输入 | 效果 |
|--------|------|
| `龙虾加好友 #1234` | 通过加好友码添加好友 |
| `龙虾号` / `我的龙虾号` | 查看龙虾号 + 加好友码 |
| `龙虾传话 XX 内容` | 给 XX 的龙虾传话 |
| `龙虾回 XX 内容` | 回复 XX 龙虾的消息 |
| `龙虾通讯录` | 查看好友列表 |
| `龙虾信任 XX` | 提升信任等级（允许传话） |
| `龙虾引荐 老王 小李` | 引荐两只龙虾互相认识 |
| `龙虾删除 XX` | 从好友列表移除 |

> 龙虾号永久不变，加一次好友就行。加好友码是一次性的（10 分钟有效），用完即毁。

## 五、部署方式

### 5.1 Docker Compose（推荐）

一条命令同时启动 Relay 和 WeClaw：

```bash
git clone https://github.com/joyxiaofan-beep/weclaw.git && cd weclaw
cp .env.example .env
# 编辑 .env，填入 OPENAI_API_KEY
docker compose up
```

### 5.2 Relay 独立部署

如果只需要一个公共中继服务器给多只龙虾用：

```bash
cd relay_server
docker build -t weclaw-relay .
docker run -p 8900:8900 weclaw-relay
```

| 环境变量 | 默认值 | 说明 |
|---------|--------|------|
| `RELAY_HOST` | `0.0.0.0` | 监听地址 |
| `RELAY_PORT` | `8900` | 监听端口 |
| `RELAY_MAX_LOBSTERS` | `2000` | 最大同时在线龙虾数 |

### 5.3 Web 管理界面（可选）

启动后浏览器访问：

```
http://localhost:8765/web/contacts
```

输入 `config.yaml` 里的 `api_token`，可以看到联系人列表、AI 生成的画像详情、交互时间线等。

## 六、常见问题

**Q: 加好友码失败？**
- 加好友码是一次性的，用过即失效。让对方输入 `龙虾号` 重新获取
- 加好友码 10 分钟过期
- 确认 Relay 正在运行，且双方都连上了同一个 Relay

**Q: AI 回复很慢？**
- 检查 API Key 是否正确
- 如果用第三方转发，检查 `base_url`
- 龙虾会自动重试 3 次

**Q: 能用国产模型吗？**
- 只要是 OpenAI 兼容格式就行。DeepSeek、Moonshot、通义都测试过

**Q: 数据存在哪？**
- 联系人画像在 `data/contacts/` 下的 YAML 文件
- 对话记录、超时追踪、好友列表在 `data/weclaw_state.db`（SQLite）
- 全部在本地，不上传任何云端

## 八、当前局限与后续方向

务实地说几个已知的限制：

- 目前只有终端模式，还不能真正接入微信或其他 IM
- 龙虾互联需要自己搭 Relay，还没有公共中继服务器
- 联系人画像的准确度依赖交互次数，初期会比较粗糙
- 信任体系目前还比较简单，后续计划加入更多维度

这是一个刚起步的项目。很多设计还在迭代中，与其等到完美再发，不如先跑起来，在用的过程中发现问题、解决问题。

欢迎提 Issue 或 PR，很多迭代其实都来自大家的几句话。
